【操作系统】4. 文件系统

文件系统

概览

文件的逻辑结构

有结构文件

由一组组相似的记录组成，又称“记录式”文件。每条记录又由若干个数据项组成，比如数据库表文件。

逻辑结构

  1. 顺序文件
    • 链式存储（不可随机存取，链式结构要遍历查找）
    • 顺序存储（可变长记录无法随机存取，定长记录可随机存取）
  2. 索引文件
    • 每个文件都有一个索引，然后建立一张索引表（定长记录的随机文件），存放文件的索引和文件地址；
  3. 索引顺序文件
    索引文件和顺序文件思想的结合。索引顺序文件中，同样会为文件建立一张索引表，但不同的是：并不是每个记录对应一个索引表项，而是一组记录对应一个索引表项。

文件目录

  1. 文件控制块FCB
    FCB的有序集合称为“文件目录”，一个FCB就是一个文件的目录项。
    FCB中包含了文件的基本信息（文件名、物理地址、逻辑结构、物理结构等），存取控制信息（是否可读可写，禁止访问的用户名单等），使用信息（如文件的建立时间、修改时间等等）
  2. 目录结构
    • 单级目录结构（整个系统只有一个目录表）——> 不支持文件重名
    • 两级目录结构（早期）：主文件目录和用户文件目录
    • 多级目录结构（树形目录结构）
    • 无环图目录结构
      用不同的文件名指向同一个文件，甚至可以指向同一个目录（共享同一目录下所有内容）
      需要为每个共享节点设置一个共享计数器，用于记录此时有多少个用户在共享该节点；
      当用户请求删除节点时，只是删除该用户的FCB，并使共享计数器减一；
  3. 索引节点
    • 传统FCB包含了文件的基本信息、存取控制信息等，但是我们在索引文件的时候只需要索引文件名，其他信息都不需要；
    • 于是可以将文件名和索引节点指针作为项的数据形成表存入存储空间，这样一块存储空间可以存放更多的表和文件索引，查找的时候就可以减少存储空间的IO次数，加快索引速度；这就是索引节点。

文件的物理结构

文件的分配方式

  1. 连续分配
    连续分配方式要求每个文件在磁盘上占有一组连续的块。
    连续分配方式的文件目录中记录存放文件的起始块号和长度，所以可以直接算出对应文件的物理块号。支持顺序访问和随机访问。

    优点：

    • 支持随机访问
    • 由于是连续存储，磁头移动时间短，顺序读写时间最快
      缺点：
    • 扩展性差，因为要确保连续
    • 存储空间利用率低，会产生难以利用的磁盘碎片

  2. 链接分配——隐式链接
    这种方式是类似链表的存储结构，目录项只包含起始块号和结束块号（相当于链头和尾）
    每次访问都从链头开始向后访问其他内容；
    所以这种方式只支持顺序访问，不支持随机访问；
    优点是拓展性好，利用率高；

  3. 链接分配——显示分配
    目录项FCB中只记录起始块号；然后在文件分配表FAT中显示记录该文件每一个物理块号对应的块地址；支持随机访问。
    优点：文件拓展方便，外存利用率高，支持随机访问，不会产生碎片；
    缺点：FAT文件分配表需要一定存储空间；
    注意：

    • 一个磁盘只设置一张FAT。开机时，将FAT读入内存，并常驻内存。
    • FAT的各个表项在物理上连续存储，且每个表项长度相同，支持随机存取（可以直接算出在哪一行）

  4. 索引分配
    系统为每个文件建立一个索引表，索引表记录了文件的各个逻辑块对应的物理块。索引表存放的磁盘称为索引块，文件数据存放的磁盘块称为数据块。（和显示链式分配很像）
    优点：文件拓展性好，支持随机访问
    缺点：索引块占用空间

  5. 混合索引
    多种索引方式的结合；

文件的存储管理

  1. 存储空间的划分与初始化
    存储空间的划分：将物理磁盘划分为一个个文件卷（逻辑卷/逻辑盘）
    存储空间的初始化：将各个文件卷划分为目录区、文件区
    1. 目录区存放文件目录信息（FCB），用于磁盘存储空间的管理信息（空闲表，位示图，超级块）
    2. 文件区就是存放具体的存储数据

存储空间管理

  1. 空闲表法
    建立一个空闲表，记录每个空闲盘的起始地址和盘块数；（适用于连续分配方式）
    如何分配连续的存储空间：首次使用算法、最佳适应算法、最坏适应算法
    如何回收磁盘块：回收时需要注意表项合并的问题（就是空闲区合并）
  2. 空闲链表法（适用于离散分布的物理结构）
    分配存储空间：若某文件申请K个盘块/盘区，则从链头开始依次摘下K个盘块，并修改空闲链的链头指针；
    回收存储空间：回收的盘块/盘区（盘区注意合并）依次挂到链尾，并修改空闲链得链尾指针；
    1. 空闲盘块链：以盘块为单位组成一条空闲链
      空闲盘块中存储着下一个空闲盘块的地址
    2. 空闲盘区链：以盘区为单位组成一条空闲链（连续的空闲盘块组成一个空闲盘区）
      空闲盘区中的第一个盘块记录了盘区的长度、下一个盘区的指针
  3. 位示图法

    位示图：每个二进制位对应一个盘块。用0或1代表空闲或已被分配；
    分配（类似将二维矩阵映射到1维）：
    1. 顺序扫描图，找到K个相邻或不相邻的“0”（代表空闲盘块）；
    2. 根据字号，块号算出对应的盘块号，将相应的盘块分配给文件；
    3. 将相应位置设为1；
      回收过程：
    4. 根据回收的盘块号计算出对应的字号，位号；
    5. 将相应的位设置为0；
  4. 成组链接法（UNIX系统）：
    空闲表法、空闲链表法不适用于大型文件系统；因为空闲表或者空闲链表可能过大；因此出现了成组链接法。
    文件卷的目录区中专门用一个磁盘块作为“超级块”，当系统启动时需要将超级块读入内存。
    空间分配：
    将空闲盘块分组，每组的第一个块记录当前空闲盘块的中的块数，第二个块记录下一组空闲盘块的首块号（链式结构），此后每一块记录存放具体数据的块号；

文件的基本操作

  1. 创建文件
    需要提供的参数：
    1. 所需要的外存大小
    2. 文件存放路径
    3. 文件名
      create系统调用：
    4. 在外存中找文件所需的空间（找空闲块）
    5. 根据文件存放路径找到该目录对应的目录文件夹，在目录中创建该文件对应的目录项，目录项中包含了文件名、文件在外存中存放位置等信息
  2. 删除文件
    delete系统调用：
    1. 找到文件名对应的目录项
    2. 根据目录项记录文件在外存的存放位置、文件大小等信息，回收文件占用的磁盘块
      需要提供的参数：
    3. 文件存放路径
    4. 文件名
  3. 打开文件
    需要的参数：
    1. 文件存放路径
    2. 文件名
    3. 需要对文件的操作类型（r, wr…）
      open系统调用：
    4. 根据文件存放路径找到目录项，并检查该用户是否有指定的操作权限；
    5. 将目录项复制到内存中的“打开文件表”中，并将表中的索引号返回给用户，之后用户使用打开文件表的编号来指明要操作的文件；（避免多次从外存中访问目录项）
    6. 打开文件表：
      系统有一张打开文件表（有打开计数器）
      每一个进程也有对应的打开文件表，并且有读写指针、访问类型等字段
  4. 关键文件
    close系统调用：
    1. 将进程的打开文件表对应的表项删除
    2. 回收分配给该文件的内存空间等资源
    3. 系统打开文件表的打开计数器-1，如果该文件的打开计数器位0，则删除对应表项
  5. 读文件
    读文件之前一定要先打开文件，此时该进程已经有表项了；
    需要的参数：
    1. 指明打开哪个文件
    2. 要读入多少数据
    3. 数据存放在内存的什么位置
      read系统调用：
      1.从打开文件表对应表项的度指针指向的外存开始，将用户指定大小的数据读入用户指定的内存区域
  6. 写文件（读文件的逆过程）
